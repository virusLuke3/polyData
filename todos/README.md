# Todo Queue

This directory tracks public future engineering work for polyData.

Private research notes and deployment journals should stay in `document/`.

## File Naming

Use this format:

```text
NNN-status-short-title.md
```

Statuses:

- `pending` - not started.
- `active` - currently being implemented.
- `blocked` - waiting on an external decision or dependency.
- `done` - completed and kept for historical context.

## Maintenance Rules

- Keep each todo focused on one architectural or product outcome.
- Include motivation, target changes, verification, and non-goals.
- Prefer adding a new todo over expanding one into a vague mega-task.
- When a todo is implemented, update its status or archive it in a follow-up cleanup.
